# ⚖️ Data Processing: The Rejection Sampler

## Overview
Even the most advanced frontier models occasionally hallucinate or fail to adhere to strict formatting constraints. If we fine-tune the Nemotron model on flawed Chain-of-Thought (CoT) data, we will actively degrade its reasoning capabilities.

To ensure **absolute data purity**, we developed the `rejection_sampler.py` script. This acts as an automated, deterministic Dual-State Judge. It evaluates every single response generated by the Teacher model against our programmatic ground truth.

## 🧠 How the Rejection Sampler Works

The `rejection_sampler.py` pipeline operates in three strict phases:

### Phase 1: Format Extraction
The script parses the raw output from the Teacher model and searches for the **last** occurrence of the `\boxed{}` tag. If the model failed to use the `<think>` tags properly or forgot the bounding box, the sample is immediately rejected.

### Phase 2: Fuzzy Normalization
Because LLMs can be unpredictable with punctuation and spacing (e.g., outputting `"Alice lives in the Blue house."` vs `"Alice lives in the blue house"`), we built a robust `normalize_answer()` function.
* It strips all domain-specific descriptive labels (like "Items:", "Max Value:", "lives in the").
* It removes all non-alphanumeric characters.
* It splits, alphabetically sorts, and concatenates the remaining strings. This ensures that an answer of `"Alpha, Beta"` is graded identically to `"Beta, Alpha"`.

### Phase 3: The Deterministic Grade
The normalized Teacher response is compared directly to the normalized Programmatic Ground Truth generated during the seed phase. 
* **Match:** The data point is deemed "Perfect" and saved to our `gold_standard` dataset for Nemotron training.
* **Mismatch:** The data point is flagged as a mathematical hallucination and discarded.

## The Result
By running our raw synthetic data through this Rejection Sampler, we guarantee a 100% accurate, hallucination-free training corpus. This high-yield, pure reasoning data is what allows our Nemotron LoRA adapter to successfully learn complex test-time compute strategies without memorizing flaws.
